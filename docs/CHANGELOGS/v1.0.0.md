## [Unreleased]

### Added
- Wallet deposit and withdrawal endpoints
- Trade execution engine (BUY/SELL) with CQRS architecture
- Pessimistic locking in `WalletService` to prevent double-spending
- Kafka integration: `trade.events` and `market.ticks` topics
- TimescaleDB hypertable migrations for `market_ticks` and `audit_logs`
- Frontend: Wallet panel UI with deposit modal
- Frontend: Trade panel UI with BUY/SELL form and order cost preview
- Kubernetes base manifests: backend Deployment, frontend Deployment, HPA

### Changed
- Upgraded `JwtAuthenticationFilter` to check Redis blacklist on every request
- Zustand store refactored into slices: `authSlice`, `marketSlice`, `portfolioSlice`

### Fixed
- `create_hypertable` error in CI by switching to `timescale/timescaledb:latest-pg16` base image
- Refresh token not invalidated on second use (replay attack window closed)

## [1.0.0] — 2026-05-26

### Added
- Initial project scaffolding: Spring Boot 3.3 + WebFlux, React 19 + Vite, PostgreSQL 16 + TimescaleDB
- User registration and login with RS256-signed JWT
- Refresh token issued in `HttpOnly; Secure; SameSite=Strict` cookie
- Logout with Redis JWT blacklist (`SET blacklist:{jti} 1 EX {ttl}`)
- Mock KYC flow: status transitions `PENDING → APPROVED`
- Role-Based Access Control: `ROLE_USER`, `ROLE_ANALYST`, `ROLE_AUDITOR`, `ROLE_ADMIN`
- Spring Security `@PreAuthorize` method-level authorization
- Flyway migrations V1–V4: `users`, `refresh_tokens`, `wallets`, `portfolio_assets`, `trades`
- Docker Compose local development environment (PostgreSQL, Kafka, Redis, backend, frontend)
- GitHub Actions CI pipeline: unit tests, JaCoCo coverage gate (≥85% backend, ≥80% frontend), Trivy image scan
- Frontend: Login / Register panel with form validation
- Frontend: Zustand store with auth slice
- Frontend: Protected routes with JWT token management
- Frontend: Sidebar navigation
- Playwright E2E: auth flows (register, login, logout, unauthenticated redirect)
- JUnit 5 + Testcontainers integration tests for Auth service

### Security
- BCrypt password hashing with cost factor 12
- RS256 JWT with 15-minute access token TTL
- TLS enforced at NGINX ingress (TLS 1.3)
- `Content-Security-Policy` header configured at NGINX
- Security headers: `X-Content-Type-Options`, `X-Frame-Options`, `Referrer-Policy`
